Autres ressources OmegaT

Module d'analyseur lexical Permet la reconnaissance des formes infléchies dans les correspondances partielles et dans les résultats du glossaire. Un guide pratique à propos du module Tokenizer est également disponible.
Module LanguageTool Vérificateur linguistique (erreurs de style, grammaire, etc.)
Module de script Permet de contrôler des fonctionnalités d'OmegaT grâce à des scripts externes. Ce module est plus évolué que la fonction d'exportation de texte permettant également l'utilisation de scripts avec OmegaT.
Module Filtres Okapi Fournit des filtres pour gérer d'autres formats de fichiers, tels que Trados TTX.
Module Apertium Intégration des fonctionnalités de traduction automatique d'Apertium avec OmegaT.

Important : l'utilisation de ces modules peut exiger l'installation d'une version d'OmegaT plus récente.

Utilitaires

Utilitaire d'alignement "Aligner" Écrit en Python Contribution de Dmitri Gabinski.
Utilitaire d'alignement "Bligner" Écrit en Python/Perl Contribution de Didier Briel.
TMXMerger Script de ligne de commande en Java pour fusionner les fichiers TMX (deux ou plus). Contribution de Henry Pijffers.
TMXCleaner Script de ligne de commande en Java permettant d'effacer les unités de traduction « superflues » dans un fichier TMX : il peut s'agir de toutes les unités de traduction sans segment cible ou dont le segment cible est identique au segment source. Contribution de Henry Pijffers.
Toxic Utilitaire destiné à la conversion des fichiers TTX (Trados TagEditor) dans un format intermédiaire permettant de les traduire avec OmegaT. Cet ensemble de scripts s'adresse aux programmeurs ayant des connaissances en tcl/tk. Les autres utilisateurs sont invités à utiliser le module de filtres Okapi. Toxic est une contribution de Marc Prior.
Toxic pour Windows Version de Toxic pouvant être exécutée sur Windows.
Toxic pour Linux Version binaire de Toxic pour Linux.
Scripts Toxic Versions script de Toxic. Multiplateforme – tcl/tk requis.
Stripper Stripper Script pour l'extraction de texte à partir de fichiers de texte brut (par ex. code source informatique) permettant la traduction avec OmegaT – tcl/tk, requis. Contribution de Marc Prior.
TKount Utilitaire de comptage de texte – tcl/tk, Abiword et pdftotext requis. Testé sur Linux. Contribution de Marc Prior.
Ondiloto ONline DIctionary LOokup TOol, utilitaire de recherche dans les dictionnaires en ligne – Linux, w3m et tcl/tk requis (ces derniers sont généralement fournis avec Linux). Contribution de Marc Prior.
Priorview Utilitaire Tcl/tk fournissant un aperçu de la mise en page formatée des fichiers traduits. Contribution de Marc Prior.
Convertisseur de fichiers TMX Wordfast Outil de conversion des mémoires de traduction au format TMX générées par le logiciel Wordfast dans une forme pouvant être lue par OmegaT. Contribution de Dmitri Gabinski.

Logiciels tiers

Logiciel tiers destiné à être utilisé avec OmegaT

Dictionnaires orthographiques Répertoire de dictionnaires d'orthographe à utiliser dans OmegaT.
OpenOffice.org
Libre Office		 Alternatives gratuites (et compatibles) à Microsoft Office.
Okapi Framework Une collection complète d'utilitaires pour préparer des fichiers pour la traduction dans un outil CAT. Permet de traduire un grand nombre de formats de fichiers qui ne peuvent être gérés de façon native dans OmegaT. Un tutoriel détaillé est disponible.
bitext2tmx Outil d'alignement GUI pour produire des fichiers de mémoire de traduction à partir de textes existants et leurs traductions.
LF Aligner Outil d'alignement intelligent en ligne de commande pour produire des fichiers de mémoire de traduction à partir de textes existants et leurs traductions.
Les scripts et les procédures de Samuel Murray Descriptions et liens vers les scripts/procédures de Samuel Murray. La plupart de ces scripts fonctionnent sous Windows, même si certaines procédures peuvent également fonctionner sur d'autres plates-formes.
Ces scripts/procédures sont disponibles pour vous permettre de :
  • Sauter des segments contenant un certain nombre de caractères
  • Extraire tout le texte source des fichiers du projet en un seul gros fichier texte
  • Créer un fichier TMX contenant tous les segments, avec source et cible identiques
  • Ajouter des termes à un glossaire tout en traduisant dans OmegaT
  • Ajouter des nombres à tous les segments pour les rendre "uniques"
  • Supprimer tout le texte sauf les balises, ou toutes les balises sauf le texte
  • Vérification orthographique (diverses procédures)
